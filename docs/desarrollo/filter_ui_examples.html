1. Chips horizontales clickeables

Más simple

Todos los valores como pills. Click toggle. Funciona bien con <10 valores por dimensión.

Pros / Cons
Pros: 0 clicks para ver opciones. Velocidad de operación máxima. Cero JS pesado.
Cons: Con 19 cursos se ve abrumador (ver fila Curso). Sin búsqueda. Sin operadores (igual / distinto / contiene).

2. Multi-select dropdowns + chips activos

Recomendado

Toolbar con dropdowns por dimensión. Cada uno abre popover con search + checkboxes. Estilo Linear/Notion.

Filtros activos: {}
Pros / Cons
Pros: Compacto independiente del N° de valores. Búsqueda inline. Chips activos visibles (estado claro). Patrón conocido (Linear, Notion, Github).
Cons: 1 click extra para abrir el dropdown (+0.5s vs chips). Más código.

Pros / Cons
Pros: Todas las opciones a la vista (sin clicks para descubrir). Búsqueda dentro de cada grupo. Bueno cuando los filtros son la actividad principal.
Cons: Roba ~25% del ancho de la pantalla siempre. Mal para dashboards con foco en visualización. Puede ser demasiado para indicadores con pocas dimensiones.

4. Faceted filters con operadores

Más potente

Estilo Notion/Airtable. Cada filtro es un chip que se construye paso a paso: dimensión → operador → valores. Soporta "es / no es / contiene / entre".

Filtros activos: {}
Pros / Cons
Pros: Más expresivo (operadores: es / no es / contiene / entre / vacío). Construcción explícita de queries. Power-user friendly.
Cons: Curva de aprendizaje (no es obvio). Más complejo de implementar (3-4× más código que opción 2). Quizá overkill para PHP.

5. Combobox global tipo Cmd-K

Power user

Una sola caja de búsqueda + chips activos. Tipeás "Curso II" y aparecen sugerencias de todas las dimensiones. Tipo command palette.

Pros: 1 sola caja para todo. Súper rápido si conocés los nombres. Look minimalista. Tipo Linear inline.
Cons: Requiere conocer las dimensiones de memoria (poco descubrible). Sin checkboxes, no se ven las opciones disponibles. Mal para usuarios nuevos.
